# GemFort: Architecture & Implementation Plan (Firecrawl Edition)

## 1. System Overview

**GemFort** needs a low-latency mobile application built to provide Sri Lankan and global gem traders with up-to-date **Exhibitions & Shows** and a continuous **Gem News Feed**.

### Tech Stack

- **Frontend:** React Native (Expo), TypeScript.
- **Backend:** Firebase (Firestore, Cloud Functions v2).
- **Web Scraper:** Firecrawl API (Bypasses bot protections, natively renders React/Angular JS, handles proxies, outputs pure Markdown).
- **AI Parser:** Google Gemini 2.5 Flash-Lite (via `@google/genai` SDK for ultra-cheap, native, and type-safe JSON extraction).

---

## 2. Pre-Determined Target URLs

To prevent unpredictable data extraction, the cloud workers will exclusively target these locked-in URLs.

### A. Gem Shows & Exhibitions Targets

These URLs are the primary authoritative sources for locking down exhibition dates (like FACETS and Gem Sri Lanka schedules).

1.  **Gem Sri Lanka (CGJTA):** `https://gslfair.com/`
2.  **FACETS Sri Lanka Authority Page:** `https://ngja.gov.lk/facet/`
3.  **International Colored Gemstone Association (ICA):** `https://www.gemstone.org/events`

### B. Global & Local Gem News Targets

These URLs provide the most rigorous and updated B2B news, trade regulations, and market trends.

1.  **Sri Lanka EDB (Export Development Board) Gem News:** `https://www.srilankabusiness.com/news/gem-and-jewellery/`
2.  **National Gem and Jewellery Authority (NGJA) Newsline:** `https://ngja.gov.lk/news-line/`
3.  **Jeweller Magazine B2B News:** `https://www.jewellermagazine.com/News/`

---

## 3. Firestore Database Schema

### Collection: `exhibitions`

- `id`: Hashed string (e.g., `base64(title+startDate)`)
- `title`: String (e.g., "Gem Exhibition 2027")
- `venue`: String (e.g., "Shangri-La Hotel, Colombo")
- `startDate`: Timestamp
- `endDate`: Timestamp
- `updatedAt`: ServerTimestamp

### Collection: `gem_news`

- `id`: Hashed string of the article URL
- `title`: String
- `source`: String (e.g., "Sri Lanka EDB")
- `summary`: String (Generated by Gemini Flash-Lite)
- `url`: String
- `publishedAt`: Timestamp

```bash

A. The Exhibitions Sync Worker
Runs every 24 hours. Firecrawl rips through any JS or bot-protection, then Gemini structures it.
import { onSchedule } from "firebase-functions/v2/scheduler";
import * as admin from "firebase-admin";
import { GoogleGenAI, Type } from "@google/genai";
import FirecrawlApp from "@mendable/firecrawl-js";

admin.initializeApp();
const db = admin.firestore();
const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });
const firecrawl = new FirecrawlApp({ apiKey: process.env.FIRECRAWL_API_KEY });

const EXHIBITION_URLS = [
  "[https://gslfair.com/](https://gslfair.com/)",
  "[https://ngja.gov.lk/facet/](https://ngja.gov.lk/facet/)",
  "[https://www.gemstone.org/events](https://www.gemstone.org/events)"
];

export const syncExhibitions = onSchedule("every day 00:00", async (event) => {
  const batch = db.batch();

  for (const url of EXHIBITION_URLS) {
    // 1. Firecrawl breaks through Cloudflare/JS and returns clean markdown
    const scrapeResult = await firecrawl.scrapeUrl(url, {
      formats: ["markdown"],
    });

    if (!scrapeResult.success) continue;

    // 2. Gemini 2.5 Flash-Lite forces a strict, cheap JSON schema output
    const response = await ai.models.generateContent({
      model: "gemini-2.5-flash-lite",
      contents: `Extract upcoming gem exhibition dates from this markdown:\n\n${scrapeResult.markdown}`,
      config: {
        responseMimeType: "application/json",
        responseSchema: {
          type: Type.ARRAY,
          items: {
            type: Type.OBJECT,
            properties: {
              title: { type: Type.STRING },
              venue: { type: Type.STRING },
              startDate: { type: Type.STRING, description: "YYYY-MM-DD" },
              endDate: { type: Type.STRING, description: "YYYY-MM-DD" },
            },
            required: ["title", "venue", "startDate", "endDate"],
          }
        }
      }
    });

    const exhibitions = JSON.parse(response.text!);

    // 3. Write securely to Firestore
    exhibitions.forEach((show: any) => {
      const docId = Buffer.from(`${show.title}-${show.startDate}`).toString("base64");
      const docRef = db.collection("exhibitions").doc(docId);
      batch.set(docRef, {
        ...show,
        startDate: admin.firestore.Timestamp.fromDate(new Date(show.startDate)),
        endDate: admin.firestore.Timestamp.fromDate(new Date(show.endDate)),
        updatedAt: admin.firestore.FieldValue.serverTimestamp(),
      }, { merge: true });
    });
  }

  await batch.commit();
});

B. The Continuous News Sync Worker
Runs every 6 hours.
import { onSchedule } from "firebase-functions/v2/scheduler";
import * as admin from "firebase-admin";
import { GoogleGenAI, Type } from "@google/genai";
import FirecrawlApp from "@mendable/firecrawl-js";

const db = admin.firestore();
const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });
const firecrawl = new FirecrawlApp({ apiKey: process.env.FIRECRAWL_API_KEY });

const NEWS_URLS = [
  { url: "[https://www.srilankabusiness.com/news/gem-and-jewellery/](https://www.srilankabusiness.com/news/gem-and-jewellery/)", source: "Sri Lanka EDB" },
  { url: "[https://ngja.gov.lk/news-line/](https://ngja.gov.lk/news-line/)", source: "NGJA Newsline" },
  { url: "[https://www.jewellermagazine.com/News/](https://www.jewellermagazine.com/News/)", source: "Jeweller Magazine" }
];

export const syncGemNews = onSchedule("every 6 hours", async (event) => {
  const batch = db.batch();

  for (const target of NEWS_URLS) {
    // 1. Firecrawl scrapes the news feed page
    const scrapeResult = await firecrawl.scrapeUrl(target.url, { formats: ["markdown"] });
    if (!scrapeResult.success) continue;

    // 2. Gemini extracts the top headlines & generates a 1-sentence summary
    const response = await ai.models.generateContent({
      model: "gemini-2.5-flash-lite",
      contents: `Extract the top 5 most recent gem industry news articles from this markdown. Write a 1-sentence summary for each.\n\n${scrapeResult.markdown}`,
      config: {
        responseMimeType: "application/json",
        responseSchema: {
          type: Type.ARRAY,
          items: {
            type: Type.OBJECT,
            properties: {
              title: { type: Type.STRING },
              summary: { type: Type.STRING },
              url: { type: Type.STRING, description: "Extract the exact hyperlink to the article" },
              publishedAt: { type: Type.STRING, description: "YYYY-MM-DD" },
            },
            required: ["title", "summary", "url", "publishedAt"],
          }
        }
      }
    });

    const articles = JSON.parse(response.text!);

    // 3. Write to Firestore
    articles.forEach((article: any) => {
      const validUrl = article.url || target.url;
      const docId = Buffer.from(validUrl).toString("base64");
      const docRef = db.collection("gem_news").doc(docId);

      batch.set(docRef, {
        title: article.title,
        source: target.source,
        summary: article.summary,
        url: validUrl,
        publishedAt: admin.firestore.Timestamp.fromDate(new Date(article.publishedAt)),
      }, { merge: true });
    });
  }

  await batch.commit();
});

5. Frontend: Expo Implementation
A. The Global & Local News Screen (Infinite Scroll)
This utilizes Firestore's pagination cursor (startAfter) to ensure fast, low-memory scrolling for your users.
import React, { useEffect, useState } from 'react';
import { View, Text, FlatList, TouchableOpacity, ActivityIndicator, Linking } from 'react-native';
import { collection, query, orderBy, limit, getDocs, startAfter } from 'firebase/firestore';
import { db } from '../config/firebase';

export default function NewsFeedScreen() {
  const [news, setNews] = useState<any[]>([]);
  const [lastDoc, setLastDoc] = useState<any>(null);
  const [loading, setLoading] = useState(false);

  const fetchNews = async (isLoadMore = false) => {
    if (loading) return;
    setLoading(true);

    let q = query(collection(db, 'gem_news'), orderBy('publishedAt', 'desc'), limit(10));

    if (isLoadMore && lastDoc) {
      q = query(collection(db, 'gem_news'), orderBy('publishedAt', 'desc'), startAfter(lastDoc), limit(10));
    }

    const snapshot = await getDocs(q);
    if (!snapshot.empty) {
      const fetchedNews = snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() }));
      setNews(isLoadMore ? [...news, ...fetchedNews] : fetchedNews);
      setLastDoc(snapshot.docs[snapshot.docs.length - 1]);
    }
    setLoading(false);
  };

  useEffect(() => {
    fetchNews();
  }, []);

  return (
    <FlatList data="{news}" keyExtractor="{(item)"> item.id}
      onEndReached={() => fetchNews(true)}
      onEndReachedThreshold={0.5}
      ListFooterComponent={loading ? <ActivityIndicator color="#0000ff" size="large"/> : null}
      renderItem={({ item }) => (
        <TouchableOpacity '#eee' 1, 16, borderBottomWidth: borderColor: onPress="{()" padding: style="{{" }}> Linking.openURL(item.url)}
        >
          <Text 'bold' 18, fontSize: fontWeight: style="{{" }}>{item.title}</Text>
          <Text '#444', 20 6, color: lineHeight: marginTop: style="{{" }}>{item.summary}</Text>
          <View 'row', 'space-between', 10 flexDirection: justifyContent: marginTop: style="{{" }}>
            <Text '#888', '600' 12, color: fontSize: fontWeight: style="{{" }}>{item.source}</Text>
            <Text '#888', 12 color: fontSize: style="{{" }}>
              {item.publishedAt?.toDate().toLocaleDateString()}
            </Text>
          </View>
        </TouchableOpacity>
      )}
    />
  );
}
```
